This project architecture allows to keep the project clean and easily findable. Whenever you need to find join, add or modify view, check 
what you could possibly write wrong in the code, it makes it possible.

The project architecture:

-project
        --dashboards : stores all LookML dashboards 
        --explores: stores all explores
          --explore: a file to set up relationship between views
        --models: stores models - connection details, formatting, explores, LookML dashboards
        --views: all datasets used for the project
            -- 1_raw: views in the exact same form as were generated by Looker (either through 'create view from table' or a derived table in SQL runner)
            -- 2_basic: detailing of the view from raw layer, detailing formatting and labelling of measures and dimensions
            -- 3_logical: newly created measures and dimensions
            --* 4_visual: html values changing visual features of measures or dimensions
      --manifest.lkml: allows to reuse the project, defining variables

--generate_lookml_views.py : script to create views from connected dataset, and put them to 1_raw and 2_basic folders with a desired content inside
